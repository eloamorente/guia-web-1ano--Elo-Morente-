Boas Práticas

Dicas essenciais para um desenvolvimento web profissional.

Semântica & Acessibilidade

  • Utilize tags semânticas: <header>, <nav>, <main>, <footer>
  • Imagens com alt descritivo
  • Labels conectadas a inputs
  • Contraste adequado entre texto e fundo
  • Ordem de tabulação lógica
<label for="email">Email</label>
<input id="email" type="email" />

Responsividade

  • Mobile-first: comece o layout pelo celular
  • Use Flexbox ou Grid para organizar
  • Unidades relativas: em, rem, %
  • Evite quebra horizontal
  • Imagens responsivas
.container { display: flex; flex-wrap: wrap; }

Organização de CSS

  • Use convenção BEM ou similar
  • Comente trechos importantes
  • Utilize variáveis CSS
  • Separe estilos em arquivos
  • Evite duplicidade de regras
:root { --color-primary: #2563EB; }

Performance leve

  • Otimize imagens (formato, tamanho)
  • Use lazy loading
  • Evite JS desnecessário
  • Minimize CSS e JS
  • Prefira fontes locais ou otimizadas
<img src="imagem.jpg" loading="lazy" alt="Descrição" />

Versionamento (Git)

  • Commits pequenos e frequentes
  • Mensagens claras e objetivas
  • Utilize branches para novas features
  • Faça pull requests para revisão
  • Documente no README
git commit -m "feat: adiciona accordion JS"

Boas práticas de JS

  • Funções puras sempre que possível
  • Nomes claros e descritivos
  • Trate erros de interação
  • Evite variáveis globais
  • Comente lógicas complexas
function soma(a, b) { return a + b; }
